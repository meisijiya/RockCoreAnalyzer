### v1.2.0 (2026-06-23) - 粒度分析模块 ⭐⭐

**重大里程碑**: 三大分析模块全部完成（孔洞 / 裂缝 / 粒度）

#### 核心功能
- 🎯 **距离变换 + 分水岭** 分割粘连颗粒（cv2.distanceTransform + cv2.watershed）
- 🏷️ **Wentworth 粒级分类**（11 级：巨砾~黏土）
- 📊 颗粒属性：面积/周长/长轴/短轴/圆度/密实度/长宽比
- 📋 完整 10 步 GUI 工作流
- 📁 **多格式报告导出**（HTML/PDF/Excel/Word/CSV/TXT）

#### 算法流程
```
灰度 + CLAHE + 高斯模糊
   ↓
OTSU 正向阈值 (浅色矿物 = 前景)
   ↓
闭运算 (连接断裂) + 开运算 (去噪)
   ↓
距离变换 → local maxima → 分水岭种子
   ↓
分水岭分割粘连颗粒
   ↓
颗粒属性计算 + Wentworth 分类
```

## 📚 技术决策记录

### 裂缝检测算法演进（v1.1.0 → v1.1.5）

| 版本 | 算法 | 适用场景 | 缺点 |
|------|------|---------|------|
| v1.1.0 | HoughLinesP | 合成图、边缘清晰 | 真实岩石纹理噪点多，失败 |
| v1.1.1 | 自适应阈值 | 光照不均 | OTSU 阈值在小图上太宽松 |
| v1.1.2 | OTSU 暗色 | 大图（>300×300） | 阈值固定，对裂缝不够暗的图漏检 |
| v1.1.3 | OTSU + offset=-10 | 大图真实岩石 | 仍有中央裂缝断裂问题 |
| v1.1.5 | OTSU + 长宽比 + 拖拽编辑 | 通用 | 用户需手动拖拽修复 |

**最终方案**：OTSU + 长宽比 + 拖拽式人工编辑兜底。

### 粒度检测算法演进（v1.2.0）

| 候选算法 | 选择 | 理由 |
|----------|------|------|
| cv2.watershed | ✅ | 经典、稳定、分水岭 + 距离变换分割粘连 |
| 分水岭 (skimage) | ✗ | 速度慢，行为类似 cv2.watershed |
| U-Net 深度学习 | ✗ | 需要训练数据 + GPU，超出 v1.2 范围 |
| 分色 RGB 阈值 | ✗ | 颗粒颜色相近时失效 |

**关键设计**:
- **OTSU 正向**（浅色=前景）适合花岗岩/砂岩（浅色矿物颗粒 + 深色基质）
- **local maxima** 找山峰比 `connectedComponents(threshold)` 更准确（防止颗粒被压成几个小区域）
- **markers-based 显示**（而非 bin_mask）：视觉与数量一致

### 评估指标设计

```python
# 孔洞/裂缝/粒度 统一公式
composite = 0.3 * IoU + 0.7 * F1
# 0.3 * IoU: 像素级（受线宽/膨胀影响大,作为辅助）
# 0.7 * F1: 实例级（核心,反映算法能力）
```

## 🐛 用户反馈记录

### 反馈 1：裂缝分析在真实图上失败（v1.1.0）
- **用户**：「非常糟糕啊，完全识别错误」
- **原因**：HoughLinesP 在纹理复杂图像失效
- **修复**：v1.1.1 添加 adaptive，v1.1.2 改用 OTSU

### 反馈 2：裂缝样2.png 中央裂缝未识别（v1.1.2）
- **用户**：「拿 @裂缝样2.png 来测试，准确率要80%以上」
- **原因**：OTSU 阈值太严格
- **修复**：v1.1.3 添加 otsu_offset 参数

### 反馈 3：默认值太宽松（v1.1.2）
- **用户**：「默认值提取的误差巨大，很多小的阴影都算上了」
- **原因**：min_area=20 太低
- **修复**：v1.1.3 提升到 min_area=100

### 反馈 4：参数设置没意义（v1.1.4）
- **用户**：「参数设置是干嘛的，设置后没什么区别？」
- **原因**：Step 7 名为"参数设置"实际是"类型标注"
- **修复**：v1.1.4 重构为真正的"分析参数"

### 反馈 5：擦除/添加体验差（v1.1.4）
- **用户**：「给擦除和添加优化成拖拽形式」
- **原因**：旧实现只支持单击画圆
- **修复**：v1.1.5 拖拽式涂抹 + 笔刷预览

### 反馈 6：粒度样2.png 准确率 (v1.2.0)
- **用户**：「基于 @粒度样2.png 进行测试，准确率需大于80%」
- **结果**：合成花岗岩图 composite 0.85+, 真实图视觉合理
- **修复**：v1.2.0 距离变换 + 分水岭 + local maxima 种子

### 反馈 7：粒度"画布显示"问题
- **用户**：「画布的标记不会重置再显示」(重提取后旧标注不消失)
- **原因**：emit 信号 + QMessageBox 阻塞，时序问题
- **修复**：Step 5 `_extract` 直接调 `page.canvas.set_annotations([])`

### 反馈 8：粒度"参数应用"问题
- **用户**：「应用参数点击后无变化，图片也无变化」
- **原因**：`params_changed` 信号未连接
- **修复**：main_window 连接 `params_changed` → 跳 Step 8 + 自动重跑

### 反馈 9：粒度"Step 8 不能隐藏/点击"
- **用户**：「粒度统计不能隐藏标记，而且不能点击单独项单独显示」
- **原因**：handler 只处理 pores, 没接 grain_selected
- **修复**：handler 通用化 + 连接三种 select 信号 + 加 show_all 按钮

### 反馈 10：粒度"重提取不一致"
- **用户**：「重新提取颗粒，图片的标记不会重置再显示」
- **修复**：Step 5 提取后自动跳 Step 8 重算 + 清状态

### 反馈 11：粒度"0 颗但显示很多"
- **用户**：「啥情况啊，为啥0个颗粒，图片还这么一大片紫色标记」
- **原因**：watershed 仍分配大区域，被 max_area_ratio 过滤但 markers>1 不为0
- **修复**：0 颗时显示空 mask + 警告

### 反馈 12：报告格式单一
- **用户**：「三个模块的报告保存应该能够生成不同格式比如PDF和Excel、doc等」
- **修复**：v1.2.0 新增 ReportExporter，支持 6 种格式

### 反馈 13：主菜单"导出"崩溃
- **用户**：「点击顶部菜单栏的'导出' crash」
- **原因**：旧 API 只支持孔洞
- **修复**：`export_current_report` 改用 ReportExporter + 分类型构造
