## 2. 核心算法

### 2.1 算法选择

考察 4 种方案:

| 方案 | 优点 | 缺点 | 决定 |
|------|------|------|------|
| cv2.watershed + distanceTransform | 经典、稳定、无需训练 | 对极密集颗粒效果一般 | ✅ 采用 |
| skimage.segmentation.watershed | API 友好 | 慢, 行为类似 cv2 | ❌ |
| U-Net 深度学习 | 效果最好 | 需 GPU + 训练数据 | ❌ 超出 v1.2 范围 |
| RGB 颜色聚类 | 简单 | 颗粒颜色相近时失效 | ❌ |

### 2.2 算法流程

```
输入: 灰度图 (CLAHE 增强)
  ↓
1. 高斯模糊 (默认 7x7) — 平滑颗粒内部纹理
  ↓
2. OTSU 正向阈值 — 浅色矿物(亮)=255 前景, 深色基质(暗)=0 背景
  ↓
3. 形态学清理:
   - 闭运算 (默认 7x7) — 连接断裂的颗粒边界
   - 开运算 (默认 3x3) — 去除小噪点
  ↓
4. 距离变换:
   sure_fg = erode(bin_mask, 3x3, 2 次)  # 去掉边缘像素
   dist = cv2.distanceTransform(sure_fg, DIST_L2, 5)
  ↓
5. 找种子 (local maxima):
   kernel = max(5, int(dist_max * 0.15))  # 自适应 kernel
   peak = (dist == dilate(dist, kernel)) & (dist > dist_max * dtr)
   dtr 默认 0.30, 30% of max — 控制种子数量
  ↓
6. 标记构造:
   markers = 1 (background)
   markers[peaks] = i + 1 (grain IDs)
  ↓
7. 分水岭:
   markers_out = cv2.watershed(bin_mask_3ch, markers)
  ↓
8. 提取颗粒 (markers > 1):
   - min_area 过滤 (默认 200 px)
   - max_area_ratio 过滤 (默认 20% — 排除大区域假阳)
   - 计算属性: 面积/周长/长轴/短轴/圆度/密实度/长宽比
   - Wentworth 粒级分类 (基于长轴 mm)
  ↓
输出: GrainAnalysisResult + markers
```

### 2.3 关键参数

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| blur_kernel | 7 | 0-25 | 高斯模糊核 (0=不模糊) |
| morph_close | 7 | 0-20 | 闭运算核 |
| morph_open | 3 | 0-10 | 开运算核 |
| min_area_px | 200 | 10-5000 | 最小候选面积 |
| max_area_ratio | 0.20 | 0-1 | 单个候选最大占比 (排除假阳) |
| distance_threshold_ratio | 0.30 | 0-1 | 距离山峰阈值 (控制种子数) |
| min_solidity | 0.40 | 0-1 | 最小密实度 |
| circularity_min | 0.20 | 0-1 | 最小圆度 |

**参数调优经验**:
- `distance_threshold_ratio` 是最敏感参数:
  - 0.20 → 445 颗 (密集小颗粒)
  - 0.30 → 282 颗 (推荐)
  - 0.50 → 30+ 颗 (中等)
  - 0.64 → 2 颗 (大区域合并)
  - 0.80 → 0 颗 (被过滤)

## 4. 准确率评估

### 4.1 评估指标

```python
composite = 0.3 * IoU + 0.7 * F1
```

- **IoU (0.3 权重)**: 像素级, 衡量前景 mask 整体匹配度
- **F1 (0.7 权重)**: 实例级, 衡量颗粒级一对一匹配
- **匹配规则**: 质心距离 ≤ 50px, IoU 优先

### 4.2 测试结果

|------|-----------|------|
| 默认合成图 (15 颗) | 0.88+ | ✅ |

## 7. 多格式报告导出 (ReportExporter)

### 7.1 设计目标

用户反馈: 「三个模块的报告保存应该能够生成不同格式比如PDF和Excel、doc等」

### 7.2 接口

```python
from rockpore.core.report_exporter import ReportExporter, SUPPORTED_FORMATS

exporter = ReportExporter(
    title="岩石粒度分析报告",
    info={"井号": "A1", "深度": "1234.5 m", ...},
    summary={"颗粒总数": "12 颗", "平均粒径": "5.2 mm", ...},
    headers=["#", "粒径(mm)", "圆度", "密实度", "粒级", ...],
    rows=[[1, 5.2, 0.85, 0.92, "细砾"], ...],
    annotated_image=bgr_ndarray,  # 可选
    notes="备注...",

# 6 种格式
exporter.export("html", "report.html")
exporter.export("pdf", "report.pdf")
exporter.export("xlsx", "report.xlsx")
exporter.export("docx", "report.docx")
exporter.export("txt", "report.txt")
exporter.export("csv", "report.csv")
```

### 7.3 三个模块共用

孔洞 / 裂缝 / 粒度 都改用 ReportExporter:

```python
# 通用 _build_exporter 方法 (main_window.py)
def _build_exporter_for_module(self, result, info, scale, image, mask, module_name):
    if hasattr(result, "pores"):
        return self._build_pore_exporter(result, info, image, mask)
    if hasattr(result, "fractures"):
        return self._build_fracture_exporter(result, info, image, mask)
    if hasattr(result, "grains"):
        return self._build_grain_exporter(result, info, scale)
    return None
```
